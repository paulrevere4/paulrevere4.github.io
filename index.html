I'm a software engineer interested in the infrastructure that systems with high demands of scalability and efficiency are built on. I'm a proud linux user and my languages of choice are Java, C/C++, and Python. In school I have taken the fundamentals like Algorithms, Data Structures, and Operating Systems and I have enjoyed special topics classes like Distributed Computing, Artificial Intelligence, and Cryptography.

Currently I am an intern at Quantcast in San Francisco, CA where I am a part of their cluster interfaces team. I work on a Java project that allows our data scientists and analysts run SQL-like queries over our enormous data stores through MapReduce jobs. Think like Presto or Hive but tailored for our systems.

For the previous school year I have been an undergraduate researcher for RPI's Scientific Computation Research Center (SCOREC) working on PUMI, a C/C++ project for the U.S. Department of Energy's FASTMathSciDAC Institute. PUMI, the Parallel Unstructured Mesh Infrastructure, is an infrastructure on which large-scale finite element analysis projects can be created. The project has tremendous demands for scalability, having been scaled to run with 256K processes communicating over MPI. I worked on a library called Attached Parallel Fields where I implemented a simple binary encoding scheme along with compression for mesh visualization files that led to a 3x reduction in their size. I also accommodated for issues that arise when writing files in massively parallel environments with thousands of CPUs. My work can be seen on the project's github page.

During the summer of 2015 I was a technical intern at the Knolls Atomic Power Laboratory in Niskayuna, NY. Knolls is a laboratory for the U.S. Department of Energy specializing in the design of nuclear reactors for the U.S. Navy. I worked on a Java project that statistically sampled nuclear data to create a representative input for a larger physics simulation. I was able to reduce the run time and memory usage of the project by orders of magnitude from what the original design of the project would have used, doing so by cutting out redundant calculations and unnecessary storage of values.
